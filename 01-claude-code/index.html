认识 Claude Code

Meet Claude Code — Your AI Coding Partner in the Terminal

Claude Code 是什么？

Claude Code 是一款运行在终端的 AI 编程助手。你只需用自然语言告诉它你想做什么，它就会帮你写代码、改代码、调试程序、运行测试，甚至管理 Git 提交。它不是一款普通的聊天机器人——它能直接操作你的文件系统、执行命令、读取和修改代码。

Claude Code is an AI coding assistant that runs in your terminal. You tell it what to do in natural language, and it writes code, fixes bugs, runs tests, and manages git commits. It's not a chatbot — it directly operates on your files, executes commands, and reads/modifies code.

💡
关键洞察 / Key Insight

这就像身边坐了一位不知疲倦的高级工程师，随时可以帮你解决任何编程问题。你不需要记住语法或命令——你只需要描述你想做什么。

It's like having a tireless senior engineer sitting next to you, ready to solve any coding problem. You don't need to memorize syntax or commands — you just describe what you want to do.

从"氛围编程"到"智能体工程"

你可能听说过"氛围编程"（Vibe Coding）——用自然语言指挥 AI 写代码。Claude Code 把这个概念提升到了一个新的层次：它不仅能写代码，还能自主研究代码库、制定计划、执行复杂任务。这就是"智能体工程"（Agentic Engineering）——让 AI 不仅仅是打字员，而是真正的工程师。

You may have heard of "Vibe Coding" — telling AI to write code in natural language. Claude Code takes this to a new level: it can research codebases, make plans, and execute complex tasks autonomously. This is "Agentic Engineering" — making AI not just a typist, but a real engineer.

命令：一键完成复杂流程

命令是你最强大的日常工具。它们以 / 开头，按下后 Claude 会按照预定义的流程来帮助你。Claude Code 有 70 多个内置命令。

Commands are your most powerful daily tool. They start with /, and when invoked, Claude follows a predefined workflow. Claude Code has 70+ built-in commands.

如果你每天都要做某件事超过一次，把它变成一个命令或技能。

If you do something more than once a day, turn it into a command or skill.

技能：自动加载的专业知识

技能是 Claude Code 的智能扩展系统。与命令不同（命令需要用户手动触发），技能会在检测到你在做特定任务时自动激活。

Skills are Claude Code's smart extension system. Unlike commands (which need manual trigger), skills auto-activate when Claude detects you're doing a specific task.

YAML 
# .claude/skills/debug/SKILL.md
---
name: debug
description: Debug failing commands or code issues
when_to_use: When the user encounters errors, bugs,
  or unexpected behavior
---

# 以下是技能的具体内容
# 告诉 Claude 如何调试的步骤和策略
中文解释

这是一个技能的定义文件，位于 .claude/skills/ 目录下。

name 是技能的名称，可以用 /debug 调用。

description 告诉 Claude 这个技能做什么，用于自动发现。

when_to_use 告诉 Claude 何时自动激活这个技能——这里是在遇到错误或意外行为时。

--- 以下是技能的实际内容，告诉 Claude 具体的调试步骤。

智能体与工作流

Agents & Workflows — Autonomous Development at Scale

智能体：自主工作者

智能体是 Claude Code 中独立运行的 AI 工作者。你可以派它去执行一个任务，它有自己独立的上下文、工具权限和模型选择。它完成后把结果交还给你。

Agents are independent AI workers in Claude Code. You can dispatch one to a task with its own isolated context, tools, permissions, and model. When done, it returns results to you.

💡
为什么用智能体？ / Why Use Agents?

每个智能体有独立的上下文窗口。这意味着主对话不会被它的研究污染，它也不会被你之前的对话混淆。这就是所谓的"计算时间"（Test Time Compute）——投入更多计算资源得到更好的结果。

Each agent has an isolated context window. This means your main conversation stays clean, and the agent isn't confused by your prior chat. This is "Test Time Compute" — investing more compute for better results.

创建自定义智能体

你可以定义自己的智能体，给它分配特定的角色、工具和模型。以下是一个"代码审查员"智能体的定义：

You can define your own agent with specific roles, tools, and models. Here's a "Code Reviewer" agent:

YAML 
# .claude/agents/code-reviewer.md
---
name: code-reviewer
description: Review code for bugs, security issues,
  and best practices
tools: Read, Grep, Glob
disallowedTools: Write, Edit, Bash
model: opus
effort: high
color: red
---

你是一个资深代码审查员。检查：
1. 安全漏洞（SQL 注入、XSS）
2. 潜在 bug 和边界情况
3. 性能和可读性问题
中文解释

给智能体取一个唯一的名字，用小写字母和连字符。

description 告诉 Claude 何时派它上场。描述越具体，Claude 判断越准确。

tools 限制它能用的工具——审查员只需要读取和搜索，不需要写文件。

disallowedTools 明确禁止某些操作——这里禁止了写入和编辑，防止它意外改代码。

model 指定用 Opus 来做深度推理，effort 设为 high 让它仔细思考。

MCP（模型上下文协议）让 Claude 能连接外部工具、数据库和 API。有了 MCP，Claude 不只是写代码——它还能操作浏览器、查询实时文档、生成架构图。

配置 MCP 服务器

JSON 
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp"]
    },
    "remote-api": {
      "type": "http",
      "url": "https://api.example.com?token=${API_KEY}"
    }
  }
}
中文解释

MCP 服务器配置在 .mcp.json 文件中。

context7 使用 npx 命令自动下载并运行，这是最常见的类型（stdio）。

remote-api 使用 http 类型，连接到远程 URL。

注意用 ${API_KEY} 环境变量引用密钥，不要把真实密钥写进文件。

CLAUDE.md：项目记忆系统

CLAUDE.md 是影响 Claude 行为最强大的文件。它告诉 Claude 你的项目约定、技术栈和偏好。

CLAUDE.md is the most powerful file for influencing Claude's behavior. It tells Claude your project conventions, tech stack, and preferences.

💡
CLAUDE.md 加载机制 / Loading Mechanism

祖先加载（向上）：Claude 从当前目录向上找，所有 CLAUDE.md 启动时立即加载。
后代加载（向下）：子目录的 CLAUDE.md 只在 Claude 进入该目录时才加载（懒加载）。

Ancestor Loading (UP): Claude walks UP from current directory, all CLAUDE.md load at startup.
Descendant Loading (DOWN): Subdirectory CLAUDE.md only loads when Claude enters that directory (lazy load).

CLAUDE.md 最佳写法

每文件控制在 200 行以内。用 <important if="..."> 标签包裹关键规则，防止 Claude 忽略。

Keep under 200 lines per file. Wrap critical rules in <important if="..."> tags to prevent Claude from ignoring them.

Git 和 PR 最佳实践

来自 Claude Code 创造者 Boris Cherny 的 Git 实践：

Git practices from Claude Code creator Boris Cherny:

小 PR Small PRs

PR 中位数 118 行——每个 PR 只做一个功能。小的 PR 更容易审查，出错更少。

PR median of 118 lines — one feature per PR. Small PRs are easier to review and have fewer bugs.

Squash Merge Squash Merge

始终使用 squash merge——保持 Git 历史干净整洁，一条线性的提交记录。

Always squash merge — keeps git history clean with a linear commit record.

频繁提交 Commit Often

至少每小时提交一次。频繁提交让你可以用 /rewind 回退到任何一个时间点。

Commit at least once per hour. Frequent commits let you /rewind to any point.

/code-review /code-review

用多智能体 PR 分析——自动检测 bug、安全漏洞和回归问题。

Multi-agent PR analysis — auto-detects bugs, security vulnerabilities, and regressions.

◈ 什么是 Hermes Agent

Hermes Agent 是一个开源的、可自我进化的 AI Agent 框架。它不仅是一个聊天机器人，更是一个具备完整工具链、记忆系统和学习能力的智能助手。

🛠

40+ 内置工具

终端执行、文件操作、浏览器自动化、代码执行、Web 搜索、图像生成、TTS 语音等

🧠

自学习闭环

完成复杂任务后自动提炼为可复用技能，技能在使用中持续自我改进

💬

多平台通信

CLI 终端 + Telegram + Discord + Slack + WhatsApp + Signal，统一对话体验

⏱

定时自动化

内置 Cron 调度器，用自然语言设置定时报告、备份、审计等自动化任务

💻

本地运行

支持 llama.cpp 本地模型，也可用 OpenRouter/OpenAI/Anthropic 等云端模型

🌐

灵活部署

6 种终端后端：本地、Docker、SSH、Daytona、Singularity、Modal

核心理念：Agent 闭环

传统 AI 聊天 = 单次问答。Hermes Agent = 观察 -> 思考 -> 使用工具 -> 学习 -> 记忆。每次交互都在积累经验，跨会话持续进化。

从零到运行只需 3 步。

  1. 启动本地模型服务器（新开一个终端标签页） 
    /Users/adam/hermes-local/start-qwen32b.sh

    等待看到 "server listening" 或类似日志表示就绪

  2. 验证模型 API 可用

  3. 启动 Hermes Agent

🔧 工具系统

Hermes 的核心能力来自丰富的工具集。Agent 根据任务自动选择合适的工具组合。

工具集 功能 需要配置 状态
terminal 核心 执行 shell 命令，支持后台进程、超时控制 无 已启用
file 核心 读写文件、搜索、补丁、目录遍历 无 已启用
vision 核心 图像理解、截图分析 无 已启用
code_execution 核心 沙箱中安全执行 Python/JS 代码 无 已启用
browser 工具 无头浏览器自动化 (Playwright) agent-browser (Node.js) 已启用
tts 工具 文字转语音 (Edge TTS 免费) 无 已启用
delegation 工具 生成子代理并行处理工作流 无 已启用
web 配置 Web 搜索/抓取 (Exa, Firecrawl, Parallel) API Key 需配置
image_gen 配置 AI 图像生成 (FAL) FAL API Key 需配置
mcp 工具 连接 MCP 服务器扩展能力 MCP 服务器配置 已启用
工具调用安全机制

Hermes 内置命令审批系统。危险操作（如 rm -rf、sudo）会暂停并请求用户确认。可通过 hermes tools 配置每个平台的工具权限。--yolo 模式可跳过确认（谨慎使用）。

Agent 驱动的记忆管理

Hermes 不是机械保存所有对话。它通过 周期性自省提示（nudge） 主动评估哪些信息值得长期记住。记忆分为事实记忆（MEMORY.md）和用户模型（USER.md），每次新会话自动加载。

记忆相关操作
# 在 Hermes 对话中
/insights --days 7       # 查看过去7天的使用分析
"记住我喜欢用 TypeScript"  # 直接告诉 Agent 要记住的事

# 管理记忆
hermes memory setup       # 配置外部记忆提供者
cat ~/.hermes/memories/MEMORY.md  # 查看记忆内容

⌨ 命令参考

Hermes 的两层命令体系：CLI 外部命令 + 对话内 Slash 命令。

CLI 外部命令 (终端中执行)

命令说明
hermes进入交互式对话
hermes --resume <id>恢复指定会话
hermes --continue恢复最近会话
hermes model选择模型/提供商
hermes setup运行完整配置向导
hermes doctor诊断问题
hermes tools配置工具启用/禁用
hermes skills list查看已安装技能
hermes gateway start启动消息网关
hermes cron list查看定时任务
hermes sessions list列出历史会话
hermes config show显示当前配置
hermes update更新到最新版本

Slash 命令 (对话中使用)

命令说明分类
/new / /reset开始新对话会话
/model [provider:model]切换模型配置
/personality [name]切换人格配置
/compress压缩上下文会话
/usage查看 token 使用量会话
/retry重新生成上一条回复会话
/undo撤销上一轮对话会话
/tools查看当前工具配置工具
/skills浏览/安装技能技能
/insights --days N使用分析报告会话
Ctrl+C中断当前执行控制

🚀 进阶功能

子代理委托

delegation 工具

Hermes 可以生成隔离的子代理并行处理多个工作流。主代理保持上下文，子代理独立执行后返回结果。适合同时处理搜索 + 编码 + 测试等多步骤任务。

MCP 集成

# 添加 MCP 服务器
hermes mcp add my-server --command "node /path/to/server.js"

# 查看已配置的 MCP 服务器
hermes mcp list

多 Profile 支持

# 创建工作用 Profile
hermes -p work             # 独立的配置/记忆/技能

# 创建个人用 Profile
hermes -p personal         # 完全隔离的另一个实例

# 管理 Profile
hermes profile list

💡 实战案例

案例 1: 项目代码审查
# 在项目目录下启动 Hermes
cd ~/my-project && hermes

# 对话：
"帮我审查 src/auth/ 目录下的代码，关注安全漏洞和性能问题"

# Hermes 会自动：
# 1. 用 file 工具读取源代码
# 2. 分析安全模式和反模式
# 3. 给出具体的改进建议和代码补丁
案例 2: 自动化数据处理
"从 data/raw/ 中的 CSV 文件提取用户行为数据，
清洗后生成可视化报告，保存到 reports/ 目录"

# Hermes 会自动：
# 1. terminal: 检查文件结构
# 2. code_execution: 运行 Python 数据处理脚本
# 3. file: 保存处理结果
# 4. 生成 HTML/图表报告
案例 3: 设置定时任务
"每天早上 9 点检查 GitHub 仓库的新 Issues，
并把摘要发到我的 Telegram"

# Hermes 会创建 Cron 任务：
# 1. 配置 GitHub API 检查
# 2. 设置 cron 表达式: 0 9 * * *
# 3. 配置 Telegram 投递
# 4. 无需手动维护，自动运行
案例 4: 网页浏览和信息提取
"打开 Hacker News 首页，总结今天排名前 5 的帖子"

# Hermes 会自动：
# 1. browser: 打开 HN 页面
# 2. vision: 分析页面内容
# 3. 提取标题、链接、讨论数
# 4. 生成结构化摘要

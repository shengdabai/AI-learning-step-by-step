核心概念

四个关键对象

Managed Agents 由四个核心原语组成。理解它们的边界和职责，是正确设计系统架构的第一步。

🤖

Agent（智能体）

持久化的配置模板，包含模型选择、系统提示词和工具集定义。创建一次，在多个 Session 中复用。类比：函数定义，而非函数调用。

持久化 · 可复用
🏗️

Environment（环境）

沙盒容器模板，定义网络访问策略、预装软件包和运行时资源。与 Agent 分离设计，允许同一 Agent 在不同环境中执行。

沙盒隔离 · 可配置
⚡

Session（会话）

Agent 的单次运行实例。绑定一个 Agent + 一个 Environment，每个任务创建独立 Session，互相隔离，状态不跨越泄漏。

一次性 · 状态隔离
📡

Events（事件流）

基于 SSE（Server-Sent Events）的双向通信机制。用户消息写入（POST），Agent 响应和工具调用流式读出（GET），通过 session.status_idle 信号任务完成。

快速开始

四步运行第一个 Agent

从零到运行中的 Agent，只需四个 API 调用。以下示例使用 Python SDK，假设你已安装依赖并配置了 API Key。

前置要求： 安装 Anthropic CLI 工具（ant）和 Python SDK，并导出 API Key。
bash — 安装依赖 
brew install anthropics/tap/ant
pip install anthropic
export ANTHROPIC_API_KEY="your-key"
1

创建 Agent（持久化配置）

Agent 是你的 AI 配置蓝图：指定模型、系统提示词和可用工具集。agent_toolset_20260401 工具集包含 Bash、文件操作和 Web 访问能力。

python 
from anthropic import Anthropic

client = Anthropic()

agent = client.beta.agents.create(
    name="Coding Assistant",
    model="claude-sonnet-4-6",
    system="You are a helpful coding assistant. Write clean, "
           "well-documented code and explain your decisions.",
    tools=[{"type": "agent_toolset_20260401"}],
)

print(f"Agent created: {agent.id}")
# Agent ID 是持久的 — 保存它以便后续复用！
2

创建 Environment（沙盒容器模板）

Environment 定义 Agent 代码执行的沙盒环境。unrestricted 网络模式允许 Agent 访问外部 URL；也可设置 none 完全隔离网络。

python 
environment = client.beta.environments.create(
    name="quickstart-env",
    config={
        "type": "cloud",
        "networking": {"type": "unrestricted"},  # 允许网络访问
    },
)

print(f"Environment created: {environment.id}")
# Environment ID 同样持久化 — 可跨 Session 复用
3

创建 Session（运行实例）

Session 将 Agent 配置与 Environment 绑定，产生一个独立的运行实例。每个任务创建独立 Session，保证状态隔离。

python 
session = client.beta.sessions.create(
    agent=agent.id,
    environment_id=environment.id,
    title="Fibonacci Task — 2026-04-09",
)

print(f"Session created: {session.id}")
# Session 是短暂的 — 每个独立任务用一个新 Session
4

发送消息并流式接收事件

使用 SSE 建立事件流连接。发送用户消息后，Agent 开始工作并实时推送事件。收到 session.status_idle 说明当前任务已完成。

python 
with client.beta.sessions.events.stream(session.id) as stream:
    # 向 Agent 发送用户消息
    client.beta.sessions.events.send(session.id, events=[{
        "type": "user.message",
        "content": [{"type": "text", "text": "创建一个 Fibonacci 数列生成脚本，"
                                              "包含单元测试，并运行验证它正确工作。"}],
    }])

    # 流式接收 Agent 响应
    for event in stream:
        match event.type:
            case "agent.message":
                # Agent 的文本回复
                print(f"🤖 Agent: {event.content}")
            case "agent.tool_use":
                # Agent 正在使用工具（如执行 bash 命令）
                print(f"🔧 工具调用: [{event.name}] {event.input}")
            case "agent.tool_result":
                # 工具执行结果
                print(f"✅ 结果: {event.content[:200]}...")
            case "session.status_idle":
                # Agent 完成任务，退出循环
                print("✨ 任务完成")
                break

API 参考

端点速查

所有端点都在 https://api.anthropic.com/v1/ 下。使用 SDK 时，Beta header managed-agents-2026-04-01 由 SDK 自动附加，无需手动处理。

端点路径 方法 描述 关键参数
/v1/agents POST 创建 Agent（持久化配置） name, model, system, tools
/v1/agents/:id GET 获取 Agent 详情 id
/v1/agents GET 列出所有 Agent limit, after_id
/v1/environments POST 创建 Environment（沙盒模板） name, config.type, config.networking
/v1/sessions POST 创建 Session（运行实例） agent, environment_id, title
/v1/sessions/:id GET 获取 Session 状态 id
/v1/sessions/:id/events GET SSE 建立 SSE 事件流，接收 Agent 实时输出 id（路径参数）
/v1/sessions/:id/events POST 向 Agent 发送用户消息 events[].type, events[].content
事件类型
事件类型 方向 触发时机
user.message → 发送 用户向 Agent 发送文本或文件
agent.message ← 接收 Agent 生成文本回复（可能分块流式）
agent.tool_use ← 接收 Agent 调用工具（bash, read_file, web_search 等）
agent.tool_result ← 接收 工具执行完毕，返回结果
session.status_idle ← 接收 Agent 完成当前任务，等待下一个用户消息
session.status_error ← 接收 Session 发生错误，需要处理
TypeScript 示例

使用 @anthropic-ai/sdk

TypeScript SDK 提供完整的类型定义，与 Python SDK 的 API 设计几乎一致。以下展示与 Python 快速开始相同的四步流程。

bash — 安装 SDK 
npm install @anthropic-ai/sdk
# 或
bun add @anthropic-ai/sdk
typescript — 完整快速开始 
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY,
});

async function runAgent() {
  // 步骤 1: 创建 Agent（一次性，可复用）
  const agent = await client.beta.agents.create({
    name: "Coding Assistant",
    model: "claude-sonnet-4-6",
    system:
      "You are a helpful coding assistant. Write clean, " +
      "well-documented code and explain your decisions.",
    tools: [{ type: "agent_toolset_20260401" }],
  });
  console.log(`Agent created: ${agent.id}`);

  // 步骤 2: 创建 Environment
  const environment = await client.beta.environments.create({
    name: "quickstart-env",
    config: {
      type: "cloud",
      networking: { type: "unrestricted" },
    },
  });

  // 步骤 3: 创建 Session
  const session = await client.beta.sessions.create({
    agent: agent.id,
    environment_id: environment.id,
    title: "Fibonacci Task",
  });

  // 步骤 4: 建立 SSE 流并发送消息
  const stream = await client.beta.sessions.events.stream(session.id);

  // 发送用户消息
  await client.beta.sessions.events.send(session.id, {
    events: [
      {
        type: "user.message",
        content: [
          {
            type: "text",
            text: "创建一个 Fibonacci 数列生成脚本，包含单元测试。",
          },
        ],
      },
    ],
  });

  // 消费事件流
  for await (const event of stream) {
    switch (event.type) {
      case "agent.message":
        console.log("🤖 Agent:", event.content);
        break;
      case "agent.tool_use":
        console.log(`🔧 工具: [${event.name}]`);
        break;
      case "agent.tool_result":
        console.log("✅ 结果:", String(event.content).slice(0, 200));
        break;
      case "session.status_idle":
        console.log("✨ 任务完成");
        stream.return(); // 关闭流
        break;
      case "session.status_error":
        console.error("❌ 错误:", event);
        stream.return();
        break;
    }
  }
}

runAgent().catch(console.error);
typescript — 带错误处理的生产级封装 
interface AgentRunOptions {
  agentId: string;       // 复用已存在的 Agent
  environmentId: string; // 复用已存在的 Environment
  prompt: string;
  onToolUse?: (name: string, input: unknown) => void;
  onMessage?: (content: string) => void;
  timeoutMs?: number;    // 默认 5 分钟
}

async function runAgentTask(opts: AgentRunOptions): Promise {
  const { agentId, environmentId, prompt, timeoutMs = 300_000 } = opts;

  const session = await client.beta.sessions.create({
    agent: agentId,
    environment_id: environmentId,
    title: `Task — ${new Date().toISOString()}`,
  });

  const controller = new AbortController();
  const timeout = setTimeout(() => controller.abort(), timeoutMs);

  try {
    const stream = await client.beta.sessions.events.stream(session.id, {
      signal: controller.signal,
    });

    await client.beta.sessions.events.send(session.id, {
      events: [{ type: "user.message", content: [{ type: "text", text: prompt }] }],
    });

    for await (const event of stream) {
      if (event.type === "agent.message") opts.onMessage?.(event.content as string);
      if (event.type === "agent.tool_use") opts.onToolUse?.(event.name, event.input);
      if (event.type === "session.status_idle") break;
      if (event.type === "session.status_error") throw new Error(`Session error: ${JSON.stringify(event)}`);
    }
  } finally {
    clearTimeout(timeout);
  }
}

  --name "DevOps Agent" \
  --model claude-sonnet-4-6 \
  --system "You are a DevOps expert." \

最佳实践

生产环境建议

以下建议来自 Managed Agents API 的设计原则，帮助你构建稳定、高效的 Agent 系统。

♻️

复用 Agent 和 Environment ID

Agent 和 Environment 是持久化的配置。创建一次后将 ID 存入数据库或环境变量，每次任务复用，只为每个任务创建新 Session。避免重复创建浪费资源。

🔧

使用 agent_toolset_20260401

这是完整工具集，包含 bash（命令执行）、文件读写、Web 访问等核心能力。如果 Agent 需要执行代码、访问网络，必须指定此工具集。

⏱️

正确处理 session.status_idle

收到此事件意味着 Agent 完成了当前任务，正在等待下一条用户消息。这是退出事件循环的正确信号，不要依赖超时来判断任务完成。

🛡️

SSE 断流重连处理

SSE 连接可能因网络波动中断。生产代码应实现重连逻辑（指数退避），并通过 GET /v1/sessions/:id 检查 Session 当前状态，决定是否需要重新订阅事件流。

🔐

Beta Header 自动附加

使用官方 Python 或 TypeScript SDK 时，managed-agents-2026-04-01 Beta header 由 SDK 自动处理，无需手动设置。直接调用 HTTP API 时才需要手动添加。

📝

精心设计 System Prompt

System prompt 对 Agent 行为影响巨大。明确指定：输出格式要求、错误处理策略（遇到失败是重试还是报告）、完成任务的判断标准、工具使用优先级。

🌐

谨慎选择网络策略

默认使用 unrestricted（允许全部网络访问）。处理敏感数据时考虑 none（完全隔离）。未来版本可能支持白名单域名策略，根据安全需求选择合适配置。

📊

监控和日志记录

记录每个 Session 的 ID 和开始时间。在 agent.tool_use 事件中记录工具名称和耗时。设置任务超时（通常 5-15 分钟），避免失控的 Agent 无限运行产生意外费用。

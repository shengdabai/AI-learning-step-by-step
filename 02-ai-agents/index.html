智能体的两种流派 — Two Schools of Agent Building

当前 Agent 构建主要分为两大流派：

🔧

流程驱动派 — Process-Driven

Dify、Coze、n8n 等低代码平台，LLM 作为数据处理后端，流程由人预先设计好

Low-code platforms like Dify, Coze, n8n — LLM acts as a data processing backend, with workflows designed by humans in advance

🧠

AI 原生派 — AI Native

真正以 AI 驱动的智能体，能自主规划、调用工具、与其他智能体协作

Truly AI-driven agents that can autonomously plan, use tools, and collaborate with other agents

💡
关键洞察 — Key Insight

流程驱动派像是"按照菜谱做菜"——每一步都规定好了；AI 原生派像是"请了一位厨师"——告诉他你想吃什么，他自己决定怎么做。

Process-driven is like "following a recipe" — every step is predetermined. AI Native is like "hiring a chef" — you tell them what you want, and they decide how to make it.

Hello-Agents 教程最独特的地方是带你从零构建一个名为 HelloAgents 的框架。下面是核心的 API 调用代码：

The most unique part of Hello-Agents is that it guides you to build your own framework called HelloAgents from scratch. Here's the core API call code:

CODE 
from openai import OpenAI

class Agent:
    def __init__(self, name, role):
        self.name = name
        self.role = role
        self.client = OpenAI()

    def respond(self, messages):
        response = self.client.chat.completions.create(
            model="gpt-4",
            messages=messages
        )
        return response.choices[0].message.content

LangGraph 教程核心代码 — LangGraph Tutorial Core Code

LangGraph 是最流行的智能体框架之一。下面是创建一个简单工作流的核心代码：

LangGraph is one of the most popular agent frameworks. Here's the core code for creating a simple workflow:

CODE 
from langgraph.graph import StateGraph, END
from typing import TypedDict

class AgentState(TypedDict):
    messages: list

workflow = StateGraph(AgentState)
workflow.add_node("research", research_agent)
workflow.add_node("write", writer_agent)
workflow.add_edge("research", "write")
workflow.set_entry_point("research")
app = workflow.compile()
中文解释

从 langgraph 导入 StateGraph，定义工作流的结构...

定义智能体的状态——这里保存消息列表...

创建一个基于状态的工作流...

添加"研究"节点：由研究智能体执行...

添加"写作"节点：由写作智能体执行...

连接两个节点：研究完成后自动进入写作...

设置入口点从研究开始...

编译工作流，变成一个可运行的应用...

💡
关键洞察 Key Insight

LangGraph 的核心思想把智能体工作流想象成一张"流程图"——每个节点是一个智能体，每条边是数据流动的方向。你只需要定义"谁做什么"和"做完之后交给谁"，框架自动处理执行和状态管理。

LangGraph's core idea is to think of agent workflows as a "flowchart" — each node is an agent, each edge is the direction of data flow. You just define "who does what" and "who gets it next," and the framework handles execution and state management automatically.

微软课程中上下文工程的关键概念——管理系统提示和对话历史：

Key concept from the Microsoft course — managing system prompts and conversation history:

CODE 
messages = [

        "role": "system",
        "content": "你是一个法律助手。请仔细分析合同条款，"
                   "并标注潜在风险。"

中文解释

user 角色消息：用户的具体请求...

让智能体分析具体的违约责任条款...

🎯
学到了吗？ Did you know?

上下文工程是智能体开发中最被低估的技能。system 提示就像给新员工的工作描述——定义了他的角色、权限和工作方式。好的 system 提示比复杂的代码更重要。

Context engineering is the most underrated skill in agent development. The system prompt is like a job description for a new employee — it defines their role, authority, and way of working. A good system prompt matters more than complex code.

Tool 01

IDE：先把工作台搭起来

IDE 解决的是"你能不能看清晰自己手上的项目"。这一步没站稳，后面都会飘。

🖥️
集成开发环境 (IDE)
你的修理台、总控台和驾驶舱

新手最缺的不是"AI 帮你写代码"，而是一个看得清全局的工作台。IDE 不是一个写代码窗口，它是你整个项目的总控中心。

左边：文件树
看清项目结构，知道每个文件在哪，入口文件是什么
上面：全局搜索
快速定位任何代码或文字，不用一个个文件翻
中间：代码编辑
直接修改代码，配合 AI 助手快速迭代
下面：终端
运行命令、启动服务器、安装依赖
推荐

Trae 国际版 / 国内版 — 对中文用户更友好，界面直观，AI 辅助编程能力强。如果你已习惯其他编辑器（VS Code、Cursor、Windsurf），也不必硬切。

很多人直接用聊天框式工具（如 Codex、Claude Code），短期很爽，长期很容易迷路。AI 给你改了 6 个文件，你连入口在哪都不知道，下一次再出 bug，就只能继续问模型，心里会越来越虚。

Tool 02

Git：版本时光机

新手学 Git 不是为了显得专业，是为了别让自己被 AI 的速度甩飞。

📦
Git 版本控制
你的版本时光机，AI 改代码快的另一面是你更容易失控

AI 改代码很快，昨天只改一处，今天模型一口气帮你动了 11 个文件。如果没有 Git，你根本说不清到底是哪一步开始坏的。

核心概念

仓库 Repository
专门放项目的仓库，不只放文件，还记住什么时候改过、改了哪些、每次改动是为了什么
本地仓库 Local
你自己电脑上的那一份，平时改代码大多数时候先在这里动手
远程仓库 Remote
放在 GitHub 等云平台上的总仓，电脑坏了它还在，别人协作也围着它转
分支 Branch
"平行施工线"。主分支像正式营业中的店面，试新功能另拉一条线，试成功再并回主线

Git 最常见的 6 个动作

clone 克隆
把远程仓库整仓复制一份到电脑里。像去总仓拿了一整套货搬回工作台
add 添加
把刚改好的东西放进待打包区。像发快递前先把要寄的东西放进箱子
commit 提交
把那箱东西封箱，贴上说明单。在本地存一个带说明的版本快照
push 推送
把本地已封箱的版本发到远程仓库。不 push 别人通常看不到你的提交
pull 拉取
把远程仓库的新变化拉回本地，合并进来。总仓更新了搬回工作台
fetch 获取
比 pull 更轻：先去总仓看有什么新货，但先不拆箱合并。看完自己决定怎么处理
最小动作链

新手先把这条链记住就够了：

clone 克隆 → 改文件 → add 添加 → commit 提交 → push 推送
Git 给你的三种底气

第一，你能看 diff —— 看这次到底改了哪几行。
第二，你能按提交回看历史。
第三，你能开分支试，不满意就退。

协作时补充：开工前先 pull；想先看更新但不合并，就用 fetch。

Tool 03

UI 设计：装修前先拉平面图

别让产品死在"脑子里很好看"。先把页面结构画清楚，后面让 AI 写前端才会更稳。

🎨
UI 设计工具
像装修前先拉一张平面图，别让施工队看着你的"感觉"盲干

很多人做 vibe-coding 的时候，代码都还没开始写，脑子里已经有了一个"应该很高级"的产品。问题是，脑子里的界面最容易骗人。

Stitch AI UI 工具
快速把"这个页面大概长什么样"压出来，AI 生成界面草图，适合快速验证想法
Pixso 设计工具
轻量、符合中文团队习惯。拖组件画界面，按钮放哪、卡片怎么排、首页先展示什么
需要回答的问题
• 按钮放哪？
• 卡片怎么排？
• 首页先展示什么？
• 用户点进去先看到什么？
Figma 备选
如果你习惯自己手绘、自己拖组件，Figma 是行业标准的传统选择，功能最全面
pixso.cn →
Tool 04

动效：产品不是只有能不能用

动效不是装饰品，它很多时候在负责反馈。这直接影响"这个产品像不像活的"。

✨
动效工具
页面能打开只是第一层，顺不顺手才是另一层

很多产品最后看起来"廉价"，不是因为功能不行，是因为所有东西都硬切。弹窗是硬弹出来的，按钮点了像没点，页面切换像瞬移。

Rive 交互式动效
做更精细、可交互的动效。可以嵌入到网页、App 中，支持状态机，能做复杂的交互动画
Lottie 轻量动画
加载动画、提示动画、成功状态动画。更轻量，适合简单的 UI 反馈场景
加载反馈
按钮按下去转一下圈，用户就知道系统在处理，不会重复点击
成功反馈
提交成功有个小动画，用户就知道事情完成了，产生满足感
关键认知

最重要的不是你一开始就做多炫，而是你要知道：动效在负责反馈。它让产品"像活的"。

rive.app → lottiefiles.com →
Tool 05

图标：AI 画的太丑了

别为了一个搜索图标折腾半小时。图标解决的是"界面里的小符号统一不统一"。

🔣
图标库
统一了产品会整洁很多，不统一用户说不出哪里怪但会觉得哪哪都散

很多新手做到这里，会开始在最不值钱的地方浪费时间。图标这块分两路走。

Iconfont 阿里巴巴矢量图标库
• 中文搜索友好
• 找图标快
• 国内常见场景都能很快搜到
• 适合中文用户使用
Lucide 开源图标库
• 干净、统一、开源
• 生态兼容度高
• 稳定、统一、不抢戏
• 适合网页产品、后台、SaaS
iconfont.cn → lucide.dev →
Tool 06

图片/视频：先当参考素材库

别在产品还没站起来的时候，先把所有精力都砸在视觉热闹上。

🖼️
图片与视频素材
能帮你提升第一眼质感，但往往不是决定生死的那一层

如果你的产品一开始核心是功能验证，那图片、视频工具先拿来做参考、找感觉、补一点演示资产就够了。

Lovart
参考素材库，找感觉、补演示资产。前期属于"加分项"不是"开工门票"
定位
像陈列和海报系统，能帮你提升第一眼质感。但项目前期，往往不是决定生死的那一层
建议

不是图和视频不重要，而是别在产品还没站起来的时候，先把所有精力都砸在视觉热闹上。先做功能，再美颜值。

Tool 07

数据库：先用 Supabase

别一上来自己盖物流园，先租一个带基础设施的小仓库就够了。

🗄️
Supabase
数据库 + 鉴权 + 存储 + 常见后端能力，打包成易用服务

数据库这个词容易把新手吓到。其实第一次做小产品，你更该关心的是：

产品要不要记住用户
用户注册登录、账号体系、个人偏好设置
要不要保存数据
聊天记录、用户创作内容、订单信息等持久化存储
为什么选 Supabase

• 把数据库、鉴权、存储、常见后端能力打包成易用服务

• 基于 PostgreSQL，开源、成熟、可扩展

Tool 08

部署：先上 Vercel

部署不是最后的仪式，它更像第一次把东西送去见人。

🚀
Vercel 部署平台
你电脑里能开只说明你自己能看，部署出去别人才打开

很多人做产品最大的幻觉："我本地已经跑起来了，差不多算做完了。"其实差很远。

为什么部署特别重要

产品一旦被部署出去，很多假问题会消失，很多真问题会冒出来：

页面错位
在别人手机上会不会错位？不同屏幕尺寸、浏览器表现是否一致
真实报错
接口会不会真报错？本地 Mock 数据和真实 API 的差异
密钥配置
你的密钥是不是漏配了？环境变量是否正确设置
这些问题
在"只活在自己电脑里"的阶段，很多都藏着。部署出来才能发现
为什么选 Vercel

• 部署快，push 代码即自动部署
• 对前端和现代 Web 项目友好
• Hobby 免费计划：做验证版、小流量版本、作品集和原型通常已经够用
• 自动 HTTPS、全球 CDN、预览部署

vercel.com →
